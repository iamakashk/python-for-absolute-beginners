# Python Virtual Environment Guide (Windows, macOS, Linux)

## What is a Virtual Environment?

A virtual environment in Python is an isolated workspace for a project.

It allows every project to have its own:

- Python packages
- Dependencies
- Package versions

This avoids conflicts between projects.

---

## Why Do We Need Virtual Environment?

Imagine:

- Project A needs Django 3
- Project B needs Django 5

Without a virtual environment, both projects may conflict.

A virtual environment keeps them separate.

---

# Step 1: Check Python Installation

## Windows

```bash
python --version
```

## macOS / Linux

```bash
python3 --version
```

---

# Step 2: Create Virtual Environment

## Windows

```bash
python -m venv myenv
```

## macOS / Linux

```bash
python3 -m venv myenv
```

### Explanation

- `python -m venv` → Uses Python's built-in virtual environment module
- `myenv` → Virtual environment folder name

You can replace `myenv` with any name.

### Example

```bash
python -m venv project_env
```

---

# Step 3: Activate Virtual Environment

## Windows CMD

```bash
myenv\Scripts\activate
```

## Windows PowerShell

```powershell
.\myenv\Scripts\Activate.ps1
```

## macOS / Linux

```bash
source myenv/bin/activate
```

---

## How to Verify It Is Activated?

After activation, terminal will look like this:

```bash
(myenv) C:\Users\Akash>
```

The environment name appears before the terminal prompt.

---

# Step 4: Install Packages Inside Virtual Environment

### Example

```bash
pip install requests
```

or

```bash
pip install flask
```

These packages will only install inside this environment.

---

# Step 5: View Installed Packages

```bash
pip list
```

---

# Step 6: Deactivate Virtual Environment

## All Operating Systems

```bash
deactivate
```

This exits the virtual environment.

---

# Project Structure Example

```text
project-folder/
│
├── myenv/
├── app.py
├── requirements.txt
```

---

# Real World Analogy

A virtual environment is like giving every Python project its own private room.

Each room has:

- Its own tools
- Its own settings
- Its own packages

Nothing gets mixed with other projects.

---

# Common Commands Summary

| Task | Windows | macOS/Linux |
|---|---|---|
| Create Environment | `python -m venv myenv` | `python3 -m venv myenv` |
| Activate Environment | `myenv\Scripts\activate` | `source myenv/bin/activate` |
| Deactivate Environment | `deactivate` | `deactivate` |

---

# Best Practices

- Create one virtual environment per project
- Do not share environments across projects
- Use `requirements.txt` for dependency management
- Keep environment names simple

---

# Export Installed Packages

```bash
pip freeze > requirements.txt
```

This stores all installed packages.

---

# Install Packages from requirements.txt

```bash
pip install -r requirements.txt
```

---

# Final Notes

Virtual environments are one of the most important concepts in Python development.

They help:

- Avoid dependency conflicts
