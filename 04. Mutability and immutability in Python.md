# Mutability vs Immutability in Python

# Introduction

One of the most important concepts in Python is:

- Mutable objects
- Immutable objects

Understanding this concept helps you understand:

- How variables work
- How memory works
- Why some objects can change
- Why some objects cannot change
- How Python handles references internally

This concept is heavily used in:

- Functions
- Lists
- Dictionaries
- Tuples
- Sets
- Object-oriented programming
- Performance optimization

---

# What is Mutability?

A mutable object can be changed after creation.

This means:

- You can modify its content
- Memory reference usually remains the same
- Original object changes directly

---

# What is Immutability?

An immutable object cannot be changed after creation.

This means:

- Any modification creates a new object
- Original object remains unchanged
- New memory location is created

---

# Simple Real-Life Analogy

## Mutable Object

A whiteboard.

You can:

- Erase
- Write again
- Modify content

Same whiteboard changes.

---

## Immutable Object

A printed paper.

If you want changes:

- You create a new paper
- Original paper stays unchanged

# Understanding Memory with id()

Python provides:

```python
id()
```

It returns memory identity of an object.

---

# Example of Immutable Object

## Integer Example

```python
x = 10

print(id(x))

x = x + 1

print(id(x))
```

## Explanation

- `10` cannot be modified
- Python creates new object `11`
- Memory address changes

---

# Output Example

```python
140734799507656
140734799507688
```

Different IDs indicate different objects.

---

# Example of Mutable Object

## List Example

```python
numbers = [1, 2, 3]

print(id(numbers))

numbers.append(4)

print(id(numbers))
```

---

# Explanation

The list changes directly.

Python does not create a new list.

Memory address remains same.

---

# Output Example

```python
2256847362240
2256847362240
```

Same IDs indicate same object modified.

# Immutable Types Detailed Explanation

# 1. int

```python
x = 5

x = x + 1
```

Python creates new integer object.

Integers are immutable.

---

# 2. float

```python
price = 99.5

price = price + 1
```

New float object is created.

---

# 3. str (String)

Strings cannot be modified directly.

## Example

```python
name = "Akash"

name[0] = "R"
```

## Output

```python
TypeError
```

---

# Correct Way

```python
name = "Akash"

new_name = "R" + name[1:]

print(new_name)
```

---

# Why Strings Are Immutable?

Benefits:

- Faster performance
- Better memory optimization
- Security
- Hashing support

Strings are heavily optimized internally.

# Function Side Effects

Mutable objects can create side effects.

## Example

```python
def add_item(data):
    data.append(100)

numbers = [1, 2, 3]

add_item(numbers)

print(numbers)
```

## Output

```python
[1, 2, 3, 100]
```

Original list changed.

---

# Immutable Objects Prevent Side Effects

## Example

```python
def add_one(x):
    x = x + 1
    return x

num = 10

add_one(num)

print(num)
```

## Output

```python
10
```

Original integer remains unchanged.

# Mutable Default Argument Problem

One of Python's famous interview questions.

---

# Wrong Example

```python
def add_item(item, my_list=[]):
    my_list.append(item)
    return my_list

print(add_item(1))
print(add_item(2))
```

## Output

```python
[1]
[1, 2]
```

Same list reused.

---

# Correct Solution

```python
def add_item(item, my_list=None):

    if my_list is None:
        my_list = []

    my_list.append(item)

    return my_list
```

# Shallow Copy vs Deep Copy

Mutability becomes important in copying objects.

---

# Shallow Copy

```python
import copy

a = [[1, 2]]

b = copy.copy(a)

b[0].append(3)

print(a)
```

## Output

```python
[[1, 2, 3]]
```

Nested object shared.

---

# Deep Copy

```python
import copy

a = [[1, 2]]

b = copy.deepcopy(a)

b[0].append(3)

print(a)
```

## Output

```python
[[1, 2]]
```

Completely separate object.

---

# Hashability and Immutability

Immutable objects are usually hashable.

Hashable objects can be used as:

- Dictionary keys
- Set elements

---

# Valid Dictionary Key

```python
data = {
    "name": "Akash"
}
```

String is immutable and hashable.

---

# Invalid Dictionary Key

```python
data = {
    [1, 2]: "numbers"
}
```

## Output

```python
TypeError
```

Lists are mutable and unhashable.

---

# Performance Perspective

Immutable objects are often:

- Faster
- Safer
- Easier to optimize

Mutable objects are:

- Flexible
- Easier to modify
- More memory efficient in updates

# Comparison Table

| Feature | Mutable | Immutable |
|---|---|---|

| Memory address changes | Usually No | Usually Yes |
| Safer | Less | More |

| Hashable | Usually No | Usually Yes |
